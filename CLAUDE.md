# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

Zwicker - marketing website and admin for a Slovak restaurant/pension ("Reštaurácia a penzión"), built on the Payload CMS website template. Front-end and admin run in the same Next.js instance.

## Payload skill

This project ships with the Payload CMS skill at `.claude/skills/payload/`. Start with `.claude/skills/payload/SKILL.md` (quick reference + security pitfalls around Local API access control, transactions, and hook loops), then dive into `.claude/skills/payload/reference/` for detailed docs on fields, hooks, access control, queries, adapters, and plugin development.

## Commands

Package manager is `pnpm` (enforced via `packageManager` field). Node >= 18.20.2 or >= 20.9.0.

- `pnpm dev` - start Next.js dev server (Payload admin at `/admin`, site at `/`)
- `pnpm build` - production build; `postbuild` also regenerates the sitemap via `next-sitemap`
- `pnpm start` - serve the production build
- `pnpm dev:prod` - clean `.next`, build, then start (for reproducing prod behavior locally)
- `pnpm lint` / `pnpm lint:fix`
- `pnpm generate:types` - regenerate `src/payload-types.ts` after schema changes (required after any collection/global/field edit)
- `pnpm generate:importmap` - regenerate `src/app/(payload)/admin/importMap.js` after adding/removing custom admin components
- `pnpm payload migrate:create` / `pnpm payload migrate` - Postgres schema migrations (see Database section)
- `pnpm test` - runs `test:int` then `test:e2e`
- `pnpm test:int` - Vitest integration tests (`tests/int/**/*.int.spec.ts`, jsdom env)
- `pnpm test:e2e` - Playwright against `tests/e2e/` (auto-starts `pnpm dev` on :3000; reuses existing server if running)
- Run a single integration test: `pnpm vitest run tests/int/<file>.int.spec.ts`
- Run a single e2e test: `pnpm exec playwright test tests/e2e/<file>.spec.ts`

## Architecture

### Two route groups in one Next.js app

- `src/app/(frontend)/` - public website. `layout.tsx` wires `draftMode`, theme providers, admin bar, header/footer. Routes include `[slug]`, `posts/`, `search/`, `(sitemaps)/`, `next/preview|exit-preview|seed`.
- `src/app/(payload)/` - Payload admin at `/admin` and REST/GraphQL API routes under `/api`. Do not edit `admin/importMap.js` by hand - it is regenerated by `pnpm generate:importmap`.

### Payload config

`src/payload.config.ts` is the entry point. Key decisions already made:

- **DB**: `@payloadcms/db-postgres` via `DATABASE_URL`. `push: true` is the adapter default in dev; set `push: false` when pointing at production and use migrations instead (see Database section).
- **Storage**: `vercelBlobStorage` (Vercel Blob) for the `media` collection, via `BLOB_READ_WRITE_TOKEN`.
- **Localization**: `sk` (default) and `en`, with fallback enabled. Many user-facing fields on `Settings` and elsewhere use `localized: true`.
- **Collections**: `Pages`, `Posts`, `Media`, `Categories`, `Users` (admin users). Pages and Posts are draft-enabled with autosave (`interval: 100` for live preview) and `schedulePublish: true`.
- **Globals**: `Header`, `Footer`, `Settings` (site settings - opening hours, contact, socials, default OG/SEO).
- **Plugins** (in `src/plugins/index.ts`): `redirectsPlugin`, `nestedDocsPlugin` (categories), `seoPlugin` (title `"... | Zwicker"`, url via `getServerSideURL()`), `vercelBlobStorage`, `formBuilderPlugin` (payment disabled), `searchPlugin` (indexes `posts`).
- **Jobs**: scheduled publish uses the Payload jobs queue. The runner endpoint authorizes either a logged-in user or `Authorization: Bearer ${CRON_SECRET}` (used by Vercel Cron). Note: Vercel's free/hobby tier is limited to daily crons.
- **CORS**: `getServerSideURL()` plus `https://$VERCEL_URL` when deployed.

### Path aliases (tsconfig)

- `@/*` -> `src/*`
- `@payload-config` -> `src/payload.config.ts` (used by `getPayload({ config: configPromise })` in server components and route handlers)

### Layout builder / blocks

Pages and Posts render their layout via `src/blocks/RenderBlocks.tsx`, which dispatches to block components under `src/blocks/` (`ArchiveBlock`, `Banner`, `CallToAction`, `Code`, `Content`, `Form`, `MediaBlock`, `RelatedPosts`). Heros live in `src/heros/` with `RenderHero.tsx` selecting among `HighImpact`, `MediumImpact`, `LowImpact`, `PostHero` based on the `hero.type` field. Adding a new block: create `config.ts` + `Component.tsx`, register the config in the collection's `blocks: [...]` array, add a case in `RenderBlocks.tsx`, then `pnpm generate:types`.

### Data fetching pattern

Server components call `getPayload({ config: configPromise })` and use the Local API. `generateStaticParams` is used for pages and posts; drafts are fetched with `overrideAccess: draft` and `draft` flag wired from `next/headers`' `draftMode()`. Wrap repeated queries inside a route with React `cache()` (see `src/app/(frontend)/[slug]/page.tsx`).

### Draft preview & live preview

- Draft preview URL is built by `src/utilities/generatePreviewPath.ts`; the admin hits `/next/preview` which enables `draftMode` and redirects to the front-end URL.
- Live preview uses `LivePreviewListener` in layouts/pages, guarded by `draft` from `draftMode()`. Autosave interval on Pages/Posts is `100ms` for responsiveness.
- `src/app/(frontend)/next/exit-preview/` disables draft mode.

### Revalidation

On-demand revalidation is driven by Payload `afterChange`/`afterDelete` hooks per collection/global (e.g. `src/collections/Pages/hooks/revalidatePage.ts`, `src/Settings/hooks/revalidateSettings.ts`, `src/hooks/revalidateRedirects.ts`). When adding new user-visible data, add matching revalidate hooks and use `req.context` flags to prevent loops (see `.claude/skills/payload/reference/HOOKS.md`).

### Access control

Shared helpers in `src/access/`: `anyone`, `authenticated`, `authenticatedOrPublished`. The pattern is: write/update/delete require `authenticated`; reads use `authenticatedOrPublished` so published docs are public while drafts stay admin-only.

### SEO

Every Pages/Posts document has a `meta` tab with `OverviewField`, `MetaTitleField` (generator), `MetaImageField`, `MetaDescriptionField`, `PreviewField`. Front-end pages call `generateMeta({ doc })` from `src/utilities/generateMeta.ts` in `generateMetadata`.

### Settings global (Slovak UI)

`src/Settings/config.ts` is organized into tabs: Otváracie hodiny, Kontakt, Sociálne siete, SEO / OG. Admin field labels are Slovak - keep that convention when extending it. `reservationEmail` is where the contact form routes reservations.

## Database & migrations

Local development defaults to `push: true` (schema sync without migrations). When pointed at a shared/production DB:

1. Set `push: false` in the adapter.
2. `pnpm payload migrate:create` - generates SQL + config snapshot.
3. Commit the generated migration files.
4. On deploy, run `pnpm payload migrate` before `pnpm start`.

Missing this step risks schema drift and data loss. See `.claude/skills/payload/reference/ADAPTERS.md` for transaction threading (`req`) when writing hooks.

## Styling

TailwindCSS v4 (via `@tailwindcss/postcss`) with `tw-animate-css`. shadcn/ui components live under `src/components/ui/` (see `components.json`). Global styles in `src/app/(frontend)/globals.css`. Payload admin-panel overrides in `src/app/(payload)/custom.scss`. `geist` is the default font.

## Environment variables

Copy `.env.example` to `.env`. Required: `DATABASE_URL`, `PAYLOAD_SECRET`. Used when present: `BLOB_READ_WRITE_TOKEN` (Vercel Blob), `CRON_SECRET` (jobs auth), `VERCEL_URL` / `VERCEL_PROJECT_PRODUCTION_URL` (auto-set on Vercel), `PREVIEW_SECRET` (draft preview), `NEXT_PUBLIC_SERVER_URL`.

## Testing conventions

- Integration tests: `tests/int/**/*.int.spec.ts` with Vitest + jsdom. `tsconfigPaths()` is loaded so `@/` imports work.
- E2E: Playwright, Chromium only, HTML reporter. `webServer.reuseExistingServer: true` so running `pnpm dev` first is fine.
- Fixtures/helpers live in `tests/helpers/`.

## Conventions

- Commits follow Conventional Commits (`feat:`, `fix:`, `refactor:`, `docs:`, `test:`, `chore:`, `ci:`, `build:`).
- `src/payload-types.ts` and `src/app/(payload)/admin/importMap.js` are generated files - regenerate, do not hand-edit. They are ignored by ESLint.
- Keep admin UI labels in Slovak when extending the Settings global or other content-owner-facing collections; front-end strings should respect the `sk`/`en` locales.
